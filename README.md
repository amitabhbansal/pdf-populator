# PDF Population Engine

Automatically fill values into **scanned (image-based) PDF forms** that have no
editable fields — without OCR or LLMs.

The forms are fixed templates, so each field's position is calibrated once and saved,
then values are drawn on top of the scanned pages at those positions. (Design details
are in `docs/`.)

## Setup

**Prerequisites:** Python 3.10+ and a desktop environment (the calibrator opens a GUI
window). No GPU or internet needed.

```bash
python -m venv .venv
source .venv/bin/activate            # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

### Quick check

The repo ships with a sample form that is already calibrated, so you can confirm the
setup works right away — just run the renderer:

```bash
python src/pdf_renderer.py
```

This writes `output/filled_<template>.pdf`. Open it to see the sample data placed on
the form. (Run all commands from the project root.)

## Configuration

Edit **`src/settings.py`** — the single place for settings (which PDF, data file, DPI,
text style). Output paths are derived automatically.

You provide one JSON file — the **data file** (e.g. `input/input_data.json`) with the
values to fill. Its keys define the fields; the calibrator uses the same keys, so
there is no separate field list to maintain:

```json
{
    "customer_name": "Amitabh Bansal",
    "address": "42, MG Road, Sector 5",
    "savings_account": true
}
```

A **`true`/`false`** value is treated as a **checkbox** (a tick is drawn when `true`,
nothing when `false`). Any other value is drawn as text.

## Usage

Run from the project root, in order:

### 1. Render the PDF pages to images
```bash
python src/pdf_to_images.py
```

### 2. Calibrate field positions (one-time per form)
```bash
python src/coordinate_calibrator.py
```
A window shows each page (needs a desktop display). **Click the window once so it has
keyboard focus**, select a field, then click where it should go. Controls:

| Key | Action |
|---|---|
| **click** | Place the current field on the visible page (click again to move it) |
| **N** / **P** | Next / previous field |
| **]** / **[** | Next / previous page |
| **R** | Reset all placements |
| **S** | Save the mapping and exit |
| **Esc** | Quit without saving |

For a checkbox field, click the **centre** of the box.

### 3. Fill the form
```bash
python src/pdf_renderer.py           # writes output/filled_<template>.pdf
```

## Adding a new form

1. Put the scanned PDF in `input/`.
2. Point `INPUT_PDF` and `DATA_FILE` in `src/settings.py` at it.
3. Run the three steps above — no code changes needed.

## Project structure

```
pdf-populator/
├── config/    mappings/<template>.json (generated by the calibrator)
├── input/     source PDFs + data JSON
├── output/    page images + filled PDFs (generated)
├── samples/   sample input data + example filled output PDFs
├── src/       settings.py, pdf_to_images.py, coordinate_calibrator.py, pdf_renderer.py
└── docs/      design & decisions
```
